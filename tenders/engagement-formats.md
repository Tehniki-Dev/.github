# Engagement Formats

TEHNIKI can enter larger-scope work through several structured formats.

## Audit-First

Used when the system state is unclear, risk is high, or scope cannot be priced honestly without technical diagnosis.

Typical outcome:

- technical assessment
- problem framing
- risk map
- recommended next phase

## Discovery and Architecture

Used when a product direction exists, but the system shape and implementation path still need structure.

Typical outcome:

- architecture direction
- phased implementation plan
- scope boundaries
- technical priorities

## Phase-Based Delivery

Used when the project can be divided into controlled stages with clear deliverables and review points.

Typical outcome:

- milestones
- visible execution stages
- reduced delivery chaos
- easier budget and priority control

## Long-Term Technical Partner Model

Used when the product is expected to evolve across multiple phases and needs continuity rather than one-off execution.

Typical outcome:

- stronger ownership
- predictable engineering rhythm
- safer long-term product development
